7. Improvement Recommendations

HIGH WebSocket Concurrency Limits
Issue: Multi-stream endpoint returns "too many concurrent TTS generations" even after 6+ seconds between requests.
Impact: Cannot benchmark persistent WebSocket (the fastest documented path).
Suggestion: Document concurrency limits per tier. Provide explicit context lifecycle management API.
MEDIUM Latency Transparency
Issue: Published 96ms TTFB measured on co-located infra; real-world is 270-360ms HTTP, ~830ms fresh WS.
Impact: Developers set unrealistic expectations from published benchmarks.
Suggestion: Publish benchmarks from multiple geographies. Add real-world latency estimates to documentation.
MEDIUM Credit Visibility
Issue: No clear indication of credit consumption per request. Ran out during testing with no warning.
Impact: Difficult to plan usage and budget for integration testing.
Suggestion: Add credit cost per request in response headers. Provide credit balance API endpoint.
MEDIUM Region Selection
Issue: No documentation on server regions or request routing.
Impact: Cannot optimize for latency by selecting nearest PoP.
Suggestion: Document available regions. Allow region pinning via request parameter or subdomain.
LOW Error Response Detail
Issue: "Insufficient credits" error does not include remaining balance or cost of failed request.
Impact: Cannot implement graceful degradation based on remaining credits.
Suggestion: Include remaining_credits and request_cost in error response body.
LOW Word Error Rate (WER)
Issue: Voice.ai WER of 0.1508 is higher than Cartesia (0.1219) and OpenAI (0.1174).
Impact: Slightly less accurate pronunciation for domain-specific text.
Suggestion: Provide pronunciation dictionary API for custom terms (e.g., "ECAPA-TDNN", "JARVIS Prime").
