HiveView — Where Does the Tab Live?

The existing HUD is a full-screen transparent overlay with Arc Reactor, transcript, and command input. Where does the Hive tab fit?

A

Segmented Control in Header

A small "Chat | Hive" toggle at the top of the HUD, below the J.A.R.V.I.S. title. Swaps the main content area between transcript and Hive feed. Arc Reactor stays visible in both modes.

Pros

  • Minimal UI change — just a toggle
  • Arc Reactor stays as visual anchor
  • One-tap switch between modes

Cons

  • Content area is shared — can't see both
  • Hive threads compete with Arc Reactor for vertical space
B

Side Panel (Slide-In)

The Hive feed slides in from the right edge of the screen as a panel overlay. The main HUD (Arc Reactor + transcript) stays visible underneath. Toggle via a "Hive" button in the menu bar or keyboard shortcut.

Pros

  • See both HUD and Hive simultaneously
  • Non-destructive — slides in/out
  • Matches PiP concept from Phase 2 spec

Cons

  • More complex layout (overlay + panel)
  • May feel cramped on smaller displays
  • Click-through behavior with panels needs care
C

Separate Window

The Hive lives in its own floating window, independent of the main HUD overlay. Toggle via menu bar or ⌘⇧H. Can be positioned anywhere, resized, minimized independently.

Pros

  • Full independence — resize, reposition, minimize
  • Maximum screen real estate for Hive content
  • Cleanest separation of concerns

Cons

  • Two windows to manage
  • Need to handle click-through for both
  • More complex window management code

Recommendation: A (Segmented Control)

Simplest to build, follows the approved Phase 1 design ("dedicated tab"), and the existing HUD already has a single-window pattern with ClickThroughWindow. Adding a segmented control is a minimal change. B and C can be Phase 2 enhancements if you want simultaneous viewing later.
