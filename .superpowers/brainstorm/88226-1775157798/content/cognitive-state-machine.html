Dynamic Cognitive State Machine

Three cognitive modes with deterministic transition triggers. Intelligence deployed only where it creates leverage.

State Transition Diagram

Event Triggers — Full Matrix

T1: BASELINE → REM

Scheduled + Conditional

  • idle_timer >= 6h
  • system_load < 30%
  • no_active_flow_state
  • episodic_memory_stale > 4h
  • graduation_candidates > 0

Budget: max 50 LLM calls per cycle

T2: → FLOW STATE

Event-Driven (any source)

  • tier2_task_decomposition
  • council_approved_build
  • critical_capability_gap
  • ouroboros_synthesis_request
  • user_triggered_build

Budget: task-scoped (deadline + token cap)

T3: → BASELINE (Spindown)

Completion or Safety

  • pr_merged or pr_rejected
  • council_session_complete
  • token_budget_exhausted
  • iron_gate_hard_reject
  • user_manual_spindown

Always returns to Baseline (safe default)

Safety Invariants

  • Default-safe: Power loss / crash → always restarts in BASELINE
  • Budget caps: REM has hard LLM call limit. Flow has deadline + token ceiling
  • Iron Gate: Flow State code never executes without AST validation
  • Human override: user_manual_spindown exits ANY state instantly
  • No state stacking: Only one cognitive state at a time (FSM, not stack)
