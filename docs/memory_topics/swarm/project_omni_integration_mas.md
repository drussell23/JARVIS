# The Omni-Integration — wiring O+V's dormant MAS into the live loop (2026-06-25)

**Trigger:** an audit found O+V's parallel-subagent machinery was BUILT but DORMANT — the Sovereign Swarm (`SwarmOrchestrator.submit`) had NO live caller; the L3 parallel-dispatch was default-OFF; **Slice 4b** (parallel-unit results feeding back into VALIDATE/GATE/APPLY) was unbuilt, so even armed fan-out produced duplicate serial work. The architecture "felt throttled" because the advanced layer was never reachable. Also confirmed: **the only always-on agency is the Venom tool loop (single-agent)**; the Swarm/L3 never fired in any of the ~17 A1 runs.

**Key architecture insight (the symbiotic split the operator envisioned):** ORCHESTRATION (worktrees + subagent processes + tool loops) runs on the HOST (M1 16GB or a 32GB GCP node) and is RAM-bound — gated by the `MemoryPressureGate` (freezes wide fan-out on 16GB, bursts on 32GB). COGNITION (each subagent's LLM generation) runs on **DoubleWord (remote, ~30× cheaper than Claude)** — so the heavy inference never touches local RAM, the M1 can do MODEST fan-out, and the concurrency bottleneck MOVES to DW throughput (bounded by BackgroundAgentPool + provider quarantine + the J-Prime failover). DW being cheap is what makes wide ephemeral fan-out economically routine.

## BUILT + MERGED (all gated default-OFF byte-identical, ~470 new tests across pieces):
1. **AST Collision Matrix** (`collision_matrix.py`, #69711) — proactive PRE-SUBMIT zero-trust gate: forbids parallel subagents on same-file OR import-coupled targets (Oracle import graph), partitions units into pairwise-disjoint fan-out + sequential remainder, default-DENY on indeterminate coupling. Emits the previously-dormant `COLLISION_FORCED_SEQUENTIAL`. Complements (not duplicates) the scheduler's reactive `_select_ready_batch` owned-paths deferral.
2. **DecomposableChaosInjector** (`chaos_injector_ast.py`, #69711) — acquires + mutates 3 MUTUALLY-ISOLATED pure-leaf fns (different files, zero import coupling verified via the graph) → forces a collision-free 3-way fan-out. N-entry manifest + revert-ALL.
3. **Per-subagent telemetry** (#69711) — worktree lifespan (create→reap) + DW inference cost (from the provider's `cost_usd`, honest-null) on `WorkUnitResult` + `[L3Telemetry]` per-unit/graph lines. (Note: the telemetry agent restructured `subagent_scheduler.execute()` to `return` from a `finally` — a code smell worth revisiting; can swallow exceptions.)
4. **DAGComposer — closes Slice 4b** (`dag_composer.py`, #69712) — map-reduces successful DISJOINT fan-out unit patches into ONE unified multi-file candidate (the collision matrix guarantees disjointness → clean UNION, not a merge) that walks VALIDATE/GATE/APPLY ONCE via the EXISTING `_apply_multi_file_candidate` batch-rollback path. Fail-CLOSED to legacy serial on any unit failure / collision-invariant violation. `JARVIS_WAVE3_DAG_COMPOSE_ENABLED`.
5. **Swarm invoker** (`swarm_invoker.py`, #69713) — wires `SwarmOrchestrator` into `SubagentScheduler`: a multi-node parallelizable DAG routes each unit through dynamic worker synthesis (`worker_synthesizer` → `SubagentFactory` cage → caged execute in worktree+sandbox); `EpistemicDeadlockBreaker` wired into the message-bus round-trip. Fail-CLOSED: NO uncaged worker ever; synthesis-fail → FAILED unit. `JARVIS_SWARM_ORCHESTRATOR_ENABLED` now has a live effect. **Golden Rule preserved (shape synthesized, no static roles).**
6. **Shadow→enforce promotion** (`shadow_enforce.py`, #69714) — REVIEW verdict → hard risk-tier/retry gate (reuses SemanticGuardian's stricter-wins escalation); PLAN ExecutionGraph DAG → authoritative, force-arms the existing fan-out (the DAG that drives the swarm). Fail-CLOSED verdict / fail-soft subsystem. `JARVIS_REVIEW_SUBAGENT_ENFORCE` + `JARVIS_PLAN_SUBAGENT_ENFORCE`.
7. **`deploy/ouroboros_omni_prod.env`** (#69715) — single activation profile: inherits the linux prod base + flips the full MAS/fan-out stack + the safe flags (conversation_bridge/last_session_summary/web_search_tier3/vision_verify/schedule_runner). DELIBERATELY EXCLUDES infra-dependent (J-Prime primacy needs a live VM) + never-built (cross-repo WRITE master lock) + side-effecty (interactive repair) flags.

**STATUS: BUILT + ARMED, NOT SOAK-PROVEN.** Every piece is gated default-OFF (byte-identical), tested individually. The COMPOSITION (all flags on together) has NEVER run live. The omni env ARMS it; the PROOF is a soak on a 32GB GCP node with a decomposable goal (the DecomposableChaosInjector's 3 isolated targets) where the fan-out actually fires → 3 parallel worktree subagents → the `[L3Telemetry]` shows lifespan + DW cost → the DAGComposer recomposes → one VALIDATE/GATE/APPLY.
