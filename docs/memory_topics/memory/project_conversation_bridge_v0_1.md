**Fact (v1.1, landed 2026-04-15 after V1 on 2026-04-15)**: ConversationBridge (`backend/core/ouroboros/governance/conversation_bridge.py`) now captures five signal sources, all untrusted, all Tier -1: (1) `tui_user` — SerpentFlow non-slash REPL lines; (2) `ask_human_q` — model's clarification question via Venom, wired in `tool_executor.py:~2268`; (3) `ask_human_a` — human's answer to ask_human; (4) `postmortem` — deterministic one-liner `postmortem op=X outcome=Y root_cause=Z` emitted from `comm_protocol.py::emit_postmortem` via `format_postmortem_payload()` (256-char cap, skipped when root_cause empty/none); (5) `voice` — reserved for V1.2. Process-global ring; multi-process deployments do NOT share (acceptable until V2). 46 tests green (23 V1 unit + 4 V1 integration + 19 V1.1).

**Why**: Derek approved D+A (`ask_human` + POSTMORTEM) as the right V1.1 pair to close the bidirectional half of the "Priority 1 conversation gap" without opening the SerpentFlow diff-stream firehose. V1 alone only closed user→O+V; the model's prior commitments (ask_human Q+A) and recent op history (postmortem) complete the conversation. Full beef-up requirements baked into plan + code: `op_id` metadata on every turn, sub-env gates (`CAPTURE_ASK_HUMAN`/`CAPTURE_POSTMORTEM`), postmortem K-cap (default 3) + TTL (default 600s), source subheaders in prompt, `record_turn` error isolation via `dropped_errors` counter. INFO log expanded to `n_user n_assistant n_postmortem` breakdown. Legacy `source="tui"` remaps to `tui_user` for one-release back-compat.

**How to apply**: **Authority invariant from v0.1 unchanged and REINFORCED in v1.1** — bridge output is consumed *only* by StrategicDirection at CONTEXT_EXPANSION. Zero authority over Iron Gate, UrgencyRouter, risk tier, policy engine, FORBIDDEN_PATH, tool protected-path checks, approval gating, or the OperationLedger body. If V1.2 / V2 / any future session proposes feeding bridge content into ROUTE, policy, ledger text, risk math, or any gate — STOP, requires new plan round with Derek.
