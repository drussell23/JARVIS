Sovereign Oracle Persistence (Phase 2) — replace the monolithic 1.4GB `codebase_graph.pkl` whose full-rewrite checkpoint caused cold-index `ControlPlaneStarvation`.

**§9 benchmark (MERGED #69543, on main):** `scripts/benchmarks/oracle_persistence_benchmark.py` raced 4 backends on a synthetic 120k-node/192k-edge DiGraph (real `NodeID/NodeData/EdgeData`). Verdict in `docs/architecture/ORACLE_PERSISTENCE_BENCHMARK_RESULTS.md`: **aiosqlite is the only backend without a fatal axis.** Chunking faces an irreducible tension — coarse shards load fast but a realistic edit-scatter touches every shard so the incremental checkpoint collapses (worst, ~680ms); fine per-file shards checkpoint cheap (15ms) but pay a +59% load tax (24k file opens). SQLite decouples read granularity (bulk SELECT) from write granularity (indexed row): ~10x cheaper checkpoint than monolithic at +38% load / 6x disk. **Caught my own benchmark bug**: measured mode C loop-lag on-loop while A/B were offloaded (bogus 12s artifact) → fixed so all four reconstruct via `asyncio.to_thread`.

**Impl (NOT merged, PR #69544, branch `oracle/sqlite-persistence`):** new `backend/core/ouroboros/oracle_persistence.py` — `PersistenceProvider` ABC + `GraphState` value object + `AioSqliteProvider` (WAL/synchronous=NORMAL/busy_timeout, cached writer conn + asyncio.Lock, BEGIN IMMEDIATE, every op aiosqlite-offloaded) + legacy `PickleProvider` (migration source) + `migrate_pickle_to_sqlite` (ingest→mark `meta.migrated_from_pkl`→ARCHIVE .pkl as `.pkl.migrated`, never delete) + corruption ladder (bounded `quick_check`→quarantine→cold rebuild) + `build_provider` factory. `TheOracle` never imports SQLite — speaks only to the ABC. Wired into `oracle.py` `_load_cache`/`_save_cache` behind `JARVIS_ORACLE_SQLITE_PERSISTENCE_ENABLED` (**default-OFF, byte-identical when off — provider never built**); shutdown closes conn. `upsert_files` = per-file dirty-replace hot path (every commit IS a checkpoint). DB path = `OracleConfig.SQLITE_DB_FILE` resolved via same `sandbox_fallback` as the pickle.

**Two bugs the test harness caught (both real):** (1) per-file dirty-replace deleted edges by the NEW node keys → the file's OLD outgoing edges survived → `add_edge` resurrected deleted nodes as stubs on reload (got 7 nodes, expected 5). Fix: delete edges by the file's OLD node keys (both directions) before replacing. (2) `database is locked` — the "reader" connection ran `CREATE TABLE` (a write) on open, colliding with a concurrent writer. Fix: `_ensure_schema_once` only on write paths → readers truly read-only (WAL 1-writer/N-reader).

24 tests in `tests/governance/test_oracle_persistence.py`, all green incl. 3 end-to-end through `TheOracle`.
