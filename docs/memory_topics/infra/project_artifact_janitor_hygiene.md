**Artifact janitor / autonomous workspace hygiene (Slices 177-178, MERGED to main 2026-06-08).** Born from the dry-run discovery that the repo accumulates ~20GB of untracked bloat (vision rust/model artifacts, `.ouroboros` session logs, venvs, model_checkpoints) vs only ~49MB git-tracked source. Closes the root cause behind the build-context bloat. Extends [[project_dw_economic_sovereignty]] (the soak this serves).

**Slice 177 — ArtifactJanitor (`artifact_janitor.py`).** Compresses aging logs (>7d → `.gz`, mtime preserved so the archive ages on schedule) + hard-prunes ancient artifacts (>30d), within WHITELISTED scan dirs only (`.ouroboros,model_checkpoints,backend/logs` via `JARVIS_ARTIFACT_SCAN_DIRS`; `os.walk` rooted at each — never source). Hard safety: age-gated (live files untouched), explicit `protect_paths` (active session) skipped, every op exception-isolated, `sweep()` NEVER raises. Gated `JARVIS_ARTIFACT_JANITOR_ENABLED` default-FALSE (§33.1 — it deletes). **FSM wiring:** `GovernedLoopService.start()` fires a DEFERRED once-per-boot sweep via `asyncio.to_thread` (off the event loop — no GIL contention), best-effort, never blocks boot, protects `self._session_dir`. Mirrors the Convergence Reaper boot pattern.

**Slice 178 — adaptive VOLUME-AWARE eviction.** Eradicates pure calendar deletion. `disk_usage_ratio()` (shutil.disk_usage on the host state volume; `JARVIS_ARTIFACT_VOLUME_PATH` or `JARVIS_STATE_DIR`/.jarvis; `usage_probe` injectable for tests). `sweep(now, *, force=False)` only evicts at/above `JARVIS_ARTIFACT_SCARCITY_THRESHOLD` (default 0.85); BELOW it, forensic logs are left FULLY INTACT regardless of age. Report carries `usage_ratio`/`scarcity_threshold`/`evicted`. **Discord telemetry:** `emit_maintenance_eviction(usage,freed,poster=)` pushes a MAINTENANCE_EVICTION via WEBHOOK (plain urllib HTTP — NO discord.py dep, works in the lean soak image; `JARVIS_DISCORD_MAINTENANCE_WEBHOOK`/`JARVIS_DISCORD_SPINE_WEBHOOK`, best-effort/no-op without one); `render_maintenance_eviction` → "🧹 Storage at 85%. Autonomous Janitor activated. 4.2GB ... compressed/cleared." GLS boot sweep emits off-thread on an actual eviction. Proven: 60%→preserved, 90%→evicted. 177=10 tests, 178=12 tests.
