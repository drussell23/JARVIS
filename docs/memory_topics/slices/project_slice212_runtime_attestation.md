**DESIGN (corrected from plan):** NOT a live origin/main check at boot — image ships no .git, and under restart:always any later merge would FALSE-TRIP and brick a healthy soak. Instead: **STAMP at build, PIN at launch.** `Dockerfile.soak`: `ARG GIT_COMMIT/GIT_DIRTY` → `/app/.build_attestation.json`, AFTER `COPY . /app` (layer-cache can't preserve stale stamp). `launch_dw_cortex_soak.sh` exports GIT_COMMIT (rev-parse HEAD) + GIT_DIRTY (status --porcelain) + pins `JARVIS_ATTESTATION_EXPECTED_COMMIT=$GIT_COMMIT`. Restarts keep the container-config pin → no false trips; relaunch re-pins. `runtime_attestation.py`: verdicts DISABLED/MATCH/MISMATCH/DIRTY_BUILD/UNSTAMPED/UNPINNED (prefix-match commits); `verify()` NEVER raises; `enforce()` → CRITICAL + best-effort sentinel webhook (`note_safety_refusal`) + raises `DeploymentIntegrityMismatch` in strict (default-ON-when-enabled). UNPINNED=warn-only (bare `docker compose up` degrades loudly not fatally). Master `JARVIS_RUNTIME_ATTESTATION_ENABLED` default-FALSE. **GLS wiring OUTSIDE the fail-soft boot block** (right after `self._state = STARTING`, before the big try) → mismatch sets FAILED + `failure_reason=deployment_integrity_mismatch` + propagates; attestation-INFRA errors swallowed (only the explicit mismatch halts).
