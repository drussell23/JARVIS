# Sovereign Telemetry Sentinel + Autopsy Protocol (2026-06-22, MERGED PR #69661)

**What:** `scripts/sovereign_sentinel.py` — a local async daemon that watches a live GCP soak node so the system monitors its own ops (no manual `gcloud ssh` tailing).
- **Async ingestion** (asyncio, NO sleep loops): `gcloud compute ssh <node> --command "sudo docker logs -f"` → async queue, exp-backoff reconnect through boot.
- **FSM-aware parsing** (env-extensible registry, no hardcoding): classifies DISPATCH (`[A1Trace]`), ADVISOR_BLOCK, DECOMPOSE, EGRESS_BLOCK (`LocalEgressOverweightError`), SOVEREIGN_YIELD, GENERATION, CONVERGENCE (orange PR/`[SOVEREIGN GRADUATION]`), FATAL. Clean event lines, not raw dump.
- **Autopsy Protocol (the Black Box):** BEFORE any `gcloud instances delete`, freeze the kill-sequence → extract last N docker-log lines + in-container FSM ledgers (`.jarvis/*.jsonl` etc via `sudo docker exec`) → local FSM-stamped `autopsy_reports/<node>_<ts>_<fsm_state>/` + manifest. Bounded by `JARVIS_SENTINEL_AUTOPSY_TIMEOUT_S` + fail-soft: the autopsy NEVER blocks the kill (billing always stops), kill proceeds even if autopsy fails (order invariant tested).
- **Good-citizen auto-kill:** boot-timeout (no first log) / stall-timeout (no FSM transition for `JARVIS_SENTINEL_STALL_TIMEOUT_S`) / fatal → autopsy then `gcloud instances delete`. Convergence → stand down (success != kill).
- **Deep lifecycle integration:** `ignite_sovereign_cloud_node.py` auto-spawns the sentinel as a detached daemon attached to the new instance (`--no-sentinel` / `JARVIS_IGNITE_SPAWN_SENTINEL` to opt out) → `autopsy_reports/sentinel_<node>.log`. 19 tests.

**INCIDENT + LESSON (load-bearing): the buggy sentinel false-killed a HEALTHY node.** The FIRST sentinel version used bare `docker logs` (no sudo). The gcloud ssh user is NOT in the docker group → `docker logs` returns a permission-denied ERROR line, which the naive watcher mistook for a "log line" (so `first_line_at` armed) while seeing ZERO real FSM transitions → at 900s it fired a FALSE stall_timeout and AUTO-KILLED `jarvis-ouroboros-soak-20260622-145847` — which was actively dispatching+blocking GOALs the whole time. **Fix: `sudo docker logs`.** Lessons: (1) a watcher that can't read the real signal is worse than no watcher — it false-positives into destruction; (2) validate the observability channel itself before trusting its verdicts; (3) the AUTOPSY exists precisely so a kill (even a wrong one) leaves a debuggable black box. Also: a `git pull --ff-only` SILENTLY fails to fast-forward when an untracked working-tree file (the old uncommitted basic script) collides with the incoming committed version — `rm` the untracked file then pull.

**LIVE CONVERGENCE BLOCKER observed (the real remaining gate, NOT yet diagnosed):** on the soak the chain reaches `[A1Trace] accept/ingest/submit (router=attached)` → `Advisor BLOCKED` (high blast + zero coverage on semantic_index.py 3247L etc.) → then **`CRITICAL [Chunking] decompose emitted 0 sub-goals -- falling back to advisor_blocked` + `[IntakeDLQ] orphaned GOAL reason=decompose_emitted_zero`** in a loop. Diagnostic: `verdict=progressing; total=2 blocked=1 ready=1 emitted=0`. So `decompose_for_block` DOES produce 2 sub-goals (test=ready + mutation=blocked) but `advance_orchestration` EMITS 0 (the I1 fallback path fires). Despite the classify_runner fix (#69646) routing BLOCK→chunking, and router=attached, the multi-step EMIT yields 0. **This is the actual thing blocking the autonomous orange PR — generation is never reached (BLOCK is pre-generation). The node burns ~no DW on these (blocked at CLASSIFY, no generation egress).** Next diagnosis: why `advance_orchestration`/multi_step emit=0 with a ready sub-goal.
