#   "build" = compile error or ABI drift
#   "infra" = toolchain missing, cmake not found, configure-stage failure

### Declarative routing table (no if-chains)

```python
@dataclass(frozen=True)
class _AdapterRule:
    pattern: re.Pattern[str]
    adapters: Tuple[str, ...]
    reason: str

# Single immutable tuple, inspected once at class load, never mutated.
_ADAPTER_RULES: Tuple[_AdapterRule, ...] = (
    _AdapterRule(
        pattern=re.compile(r"^(mlforge|bindings)/"),
        adapters=("python", "cpp"),
        reason="native sublayer: mlforge/bindings require dual verification",
    ),
    _AdapterRule(
        pattern=re.compile(r"^(reactor_core|tests)/"),
        adapters=("python",),
        reason="pure python layer",
    ),
    _AdapterRule(
        pattern=re.compile(r".*"),  # catch-all
        adapters=("python",),
        reason="default: python adapter",
    ),
)

def _route(changed_files, repo_root):
    # Union-of-all-matches across all changed files. Never first-match.
    required: set[str] = set()
    for path in changed_files:
        norm = _normalize(path, repo_root)  # raises BlockedPathError if outside-repo
        for rule in _ADAPTER_RULES:
            if rule.pattern.match(norm):
                required.update(rule.adapters)
                break  # first matching rule per file (table order), union across files
    return frozenset(required)
```

### Path normalization — hard gate, not warn-and-skip

```python
class BlockedPathError(Exception):
    # Raised when a changed file is outside the repo root. Causes CANCELLED.
    pass

def _normalize(path: Path, repo_root: Path) -> str:
    try:
        resolved = path.resolve()
        return resolved.relative_to(repo_root.resolve()).as_posix()
    except ValueError:
        raise BlockedPathError(
            f"Path {path} resolves outside repo root {repo_root}. "
            "Pipeline CANCELLED — security gate."
        )
```

### CppAdapter: per-op isolation, failure taxonomy, ABI probe

```python
class CppAdapter:
    name = "cpp"
    _CMAKE_GENERATOR = "Ninja"  # explicit, not env-dependent

    def _build_dir(self, op_id: str, sandbox_dir: Optional[Path]) -> Path:
        # Per-op dir — no shared global build/ when concurrency > 1
        base = sandbox_dir or self._scratch_root
        return base / op_id / "cpp-build"

    async def run(self, test_files, sandbox_dir, timeout_budget_s, op_id):
        build_dir = self._build_dir(op_id, sandbox_dir)
        t0 = time.monotonic()

        # Timeout split: reserve min(30s, 20% of budget) for ctest
        build_budget = timeout_budget_s * 0.8
        test_budget = timeout_budget_s - build_budget

        build_ok, build_out, build_exit_ctx = await self._cmake_build(
            build_dir, build_budget, sandbox_dir
        )
        if not build_ok:
            return AdapterResult(
                adapter="cpp", passed=False,
                failure_class=self._classify_build_failure(build_out, build_exit_ctx),
                ...
            )

        abi_ok, abi_err = await self._abi_probe(build_dir, sandbox_dir)
        if not abi_ok:
            return AdapterResult(
                adapter="cpp", passed=False,
                failure_class="build",  # ABI drift = build-class, not infra
                ...
            )

        remaining = test_budget - (time.monotonic() - t0)
        if remaining <= 0:
            return AdapterResult(adapter="cpp", passed=False, failure_class="infra", ...)

        return await self._run_ctest(build_dir, remaining, op_id)

    def _classify_build_failure(self, output: str, exit_ctx: str) -> str:
        # Infra: keyed on process exit context FIRST, then markers
        if exit_ctx in ("executable_not_found", "configure_stage"):
            return "infra"
        infra_markers = (
            "cmake: command not found", "ninja: command not found",
            "No CMAKE_CXX_COMPILER", "Could not find toolchain",
        )
        if any(m in output for m in infra_markers):
            return "infra"
        return "build"  # compile error

    async def _abi_probe(self, build_dir, sandbox_dir):
        # Derive .so path from cmake install manifest — no assumed path
        install_tmp = build_dir / "_abi_probe_install"
        # cmake --install writes install_manifest.txt listing all files
        await cmake_install(build_dir, prefix=install_tmp)
        so_files = list(install_tmp.rglob("*.so"))
        if not so_files:
            return False, "ABI probe: no .so files found in install manifest"
        for so in so_files:
            if not await can_load_shared_lib(so):
                return False, f"ABI probe: {so.name} failed to load"
        return True, ""
```

## Section 4: Error Handling + Restart Recovery + Advanced Gaps

### Boot-time reconciliation (restart mid-apply)

```
On GovernedLoopService.start():

1. Scan all op_ids in ledger for latest_state == APPLIED (orphaned):
   -> Hash-guarded rollback:
     current_hash = sha256(target_file)
     expected_hash = ledger_entry.data["post_apply_hash"]
     if current_hash != expected_hash:
         -> file drifted (human edit / new op) -> NO rollback
         -> append FAILED(reason="boot_recovery_hash_mismatch")
         -> emit ALERT(outcome="manual_intervention_required")
     else:
         -> Attempt rollback via stored RollbackArtifact
         -> If success: append FAILED(data={"reason": "boot_recovery", ...})
         -> If failure: append FAILED(data={"reason": "boot_recovery_failed", ...})
                        emit ALERT(outcome="manual_intervention_required")

   Idempotency: before any action, check ledger_entry.data for
   "recovery_attempted=True". If present: skip — already handled.
   After deciding to act: write "recovery_attempted=True, recovery_attempt_id=<uuid>"
   to ledger BEFORE executing rollback.

2. ApprovalStore.expire_stale(timeout_seconds=config.approval_ttl_s)
   -> Collect all expired op_ids
   -> Emit single ALERT batch summary (not per-op — prevents notification storm)
   -> Write per-op ledger entries for audit

3. Stale PLANNED cancellation:
   -> Age = now() - ledger_entry.timestamp
   -> Clock policy: wall-clock in ledger; skew_tolerance_s=60
      If delta implausible (negative or >1 week): log WARNING, skip
   -> If age > cold_start_grace_s (config, default 300s): append CANCELLED
```

### Failure -> terminal state mapping

| Failure | OperationState | failure_class in data | Rationale |
|---|---|---|---|
| Path outside repo | CANCELLED | "security" | Hard gate, not code problem |
| Test failure | FAILED | "test" | Code failed, rollback applied |
| Compile error | FAILED | "build" | Code failed, rollback applied |
| Toolchain missing | FAILED | "infra" | Env problem; no rollback |
| Approval timeout/rejected | CANCELLED | "approval" | Human decision or TTL |
| Approve after terminal | audit-only (ApprovalStore) | — | No state mutation |
| Budget exhausted | CANCELLED | "timeout" | Time contract violated |
| Rollback self fails | FAILED | "rollback_failed" | Manual intervention required |

Note: POSTMORTEM is not a ledger state — represented as FAILED with failure_class=infra|rollback_failed.
SUPERSEDED is ApprovalStore audit-only; never written to ledger.

### Backpressure on notification fanout

```python
_QUEUE_CAPACITY = 128
_RESERVED_SLOTS = 16   # hard reservation for DECISION/ALERT

_DROP_PRIORITY = {
    MessageType.HEARTBEAT: 0,   # drop first (silent)
    MessageType.INTENT: 1,      # evict if needed
    MessageType.DECISION: 2,    # never evicted — reserved lane
    MessageType.ALERT: 3,       # never evicted — reserved lane
}

# Invariant: DECISION/ALERT always have _RESERVED_SLOTS available.
# Non-critical messages can only use slots 0..(QUEUE_CAPACITY - RESERVED_SLOTS - 1).
```

### Flaky test discrimination policy

```
TestResult.flake_suspected == True: first run failed, exactly 1 retry, retry passed.

Retry contract (deterministic):
  - Retry count: exactly 1 (no more)
  - Retry delay: 0s (immediate)
  - Retry env: subprocess launched with clean env (no inherited PYTEST_* vars)
  - PYTHONHASHSEED=0 for reproducibility
  - Per-adapter independent retry (Python retries independently of C++)

Pipeline policy:
  flake_suspected=True + SAFE_AUTO   -> escalate to APPROVE phase
  flake_suspected=True + REVIEW_REQUIRED -> already in APPROVE, no change
  flake_suspected=True + BLOCKED     -> still BLOCKED, flake irrelevant
  flake_suspected=True never treated as clean pass.
  Recorded in ReadyToCommitPayload.
```

## Accepted Constraints Summary

### State vocabulary (canonical, no invented states)
- OperationState enum: PLANNED, APPLIED, CANCELLED, FAILED, ROLLED_BACK (existing only)
- failure_class in ledger data: "test", "build", "infra", "security", "approval", "timeout", "rollback_failed"
- SUPERSEDED: ApprovalStore audit row only — never a ledger state
- POSTMORTEM: not a state — represented as FAILED with failure_class=infra|rollback_failed

### B-compatible seam points
- _build_comm_protocol(config=None) — accepts config now, uses env/defaults; ready for GovernanceConfig
- TestRunner.__init__(repo_root, timeout, adapters=None) — adapters injectable for testing
- Supervisor subcommands registered via _register_self_dev_commands(parser, integration_fn) — integration_fn injectable

### Advanced gaps addressed
- Restart mid-apply: hash-guarded, idempotent (recovery_attempted marker), audit-trailed
- Cross-process races: file lock on op_id + ledger idempotency check
- Flaky test discrimination: deterministic 1-retry policy, clean env, SAFE_AUTO escalation
- Native ABI drift: install-manifest-driven probe after every C++ build
- Backpressure: bounded queue (128 cap), 16-slot hard reservation for DECISION/ALERT
- Contract-aware adapter selection: declarative table, union-of-all-matches, no if-chains
